# Storyworks 人物卡设计计划

> 制定日期：2026-06-26
> 目标：构建专业级人物卡系统，支持双视角模式、AI分步生成、关系图谱管理

---

## 一、功能需求汇总

### 1.1 用户选择确认
| 功能点 | 选择 |
|--------|------|
| 属性系统 | 8个核心分类 + 支持用户自定义分类 |
| 属性值类型 | 基础类型：文本、数字、布尔 |
| 属性继承 | 支持从已有角色复制属性结构 |
| developer_mode | 8个分类完整展示 |
| player_mode | 只显示 basic + knowledge |
| player_mode 显示 | 可配置（对每个字段单独设置显示方式） |
| 关系类型 | 预设类型 + 自定义类型 |
| 关系方向 | 支持双向和单向混合 |
| 关系详情 | 类型 + 描述 + 数值 + 历史事件 |
| 关系图谱 | 交互式图谱（可拖拽、点击、筛选）、可编辑 |
| 世界书关联 | AI自动推荐 + 手动选择 |
| 关联展示 | 双向显示 |
| 生成方式 | 分步生成 + 支持迭代 |
| 生成参数 | 角色类型、概念、详细程度、世界观参考、风格 |
| AI质量检查 | 可选 |
| 列表展示 | 卡片视图 |
| 搜索筛选 | 类型、标签、状态、全文搜索 |
| 批量操作 | 删除、导出、应用模板 |
| 导出格式 | JSON、Markdown |

## 二、数据结构设计

### 2.1 角色核心数据结构

```typescript
interface Character {
  id: string;
  projectId: string;
  name: string;
  characterType: 'protagonist' | 'supporting' | 'antagonist' | 'npc';
  
  // 开发者模式完整数据
  developerData: {
    basic: CharacterBasic;
    knowledge: CharacterKnowledge;
    secrets: CharacterSecrets;
    attributes: CharacterAttributes;
    relations: CharacterRelation[];
    inventory: CharacterItem[];
    skills: CharacterSkill[];
    fortune: CharacterFortune;
    [customCategory: string]: any;  // 自定义分类
  };
  
  // 玩家模式数据（精简）
  playerData: {
    basic: CharacterBasic;
    knowledge: CharacterKnowledge;
    // secrets/fortune 等显示为"未知"或可配置内容
    [customCategory: string]: any;
  };
  
  // 字段可见性配置
  fieldVisibility: FieldVisibilityConfig;
  
  // 关联的世界书条目
  worldEntryIds: string[];
  
  // 元数据
  tags: string[];
  createdAt: Date;
  updatedAt: Date;
  
  // AI生成相关
  aiGenerated: boolean;
  generationHistory: GenerationStep[];
}

// 基本信息
interface CharacterBasic {
  name: string;
  identity: string;
  level: string;
  summary: string;
}

// 认知信息
interface CharacterKnowledge {
  personality: string;
  appearance: string;
  background: string;
  motivation: string;
}

// 秘密信息
interface CharacterSecrets {
  trauma: string;
  hiddenIdentity: string;
  secret: string;
}

// 属性信息
interface CharacterAttributes {
  health: string;
  special: string;
}

// 关系
interface CharacterRelation {
  targetId: string;
  targetName: string;
  relationType: string;      // 预设或自定义类型
  direction: 'to' | 'from' | 'bidirectional';
  description: string;
  numericValue?: number;     // 数值（好感度/信任度等）
  events: RelationEvent[];   // 关联历史事件
}

interface RelationEvent {
  id: string;
  title: string;
  description: string;
  timestamp: string;         // 故事内时间
}

// 物品
interface CharacterItem {
  name: string;
  type: string;
  description: string;
}

// 技能
interface CharacterSkill {
  name: string;
  level: string;
  description: string;
}

// 命运
interface CharacterFortune {
  destinyTags: string[];
  causalHints: string[];
}

// 字段可见性配置
interface FieldVisibilityConfig {
  // 每个分类的每个字段都可以单独配置
  [category: string]: {
    [field: string]: {
      visible: boolean;           // 在player_mode是否可见
      displayMode: 'full' | 'partial' | 'hidden' | 'custom';
      customDisplay?: string;     // 自定义显示文本（如"似乎有不可告人的过去"）
    };
  };
}
```

### 2.3 角色模板系统

```typescript
interface CharacterTemplate {
  id: string;
  name: string;
  description: string;
  characterType: string;
  
  // 分类结构定义
  categories: CategoryDefinition[];
  
  // 默认字段值
  defaultValues: Record<string, any>;
  
  // 字段可见性默认配置
  defaultVisibility: FieldVisibilityConfig;
}

interface CategoryDefinition {
  id: string;
  name: string;
  icon?: string;
  fields: FieldDefinition[];
  isCustom: boolean;        // 是否为用户自定义分类
}

interface FieldDefinition {
  id: string;
  name: string;
  type: 'text' | 'number' | 'boolean';
  required: boolean;
  defaultValue?: any;
  placeholder?: string;
}
```

## 四、功能详细设计

### 4.1 属性系统

#### 4.1.1 核心分类
系统预设8个分类，每个分类包含固定字段：

| 分类 | 字段 | 说明 |
|------|------|------|
| basic | name, identity, level, summary | 基本信息 |
| knowledge | personality, appearance, background, motivation | 认知信息 |
| secrets | trauma, hiddenIdentity, secret | 秘密信息 |
| attributes | health, special | 属性信息 |
| relations | targetId, relationType, description, numericValue, events | 关系列表 |
| inventory | name, type, description | 物品列表 |
| skills | name, level, description | 技能列表 |
| fortune | destinyTags, causalHints | 命运信息 |

#### 4.1.2 自定义分类
- 用户可以添加自定义分类（如"战斗属性"、"魔法系统"、"阵营声望"等）
- 自定义分类支持添加字段（文本、数字、布尔类型）
- 自定义分类可以设置图标和排序

#### 4.1.3 模板系统
- 预设角色模板（主角模板、NPC模板等）
- 可从已有角色复制属性结构
- 模板包含分类结构和默认值

### 4.2 双视角模式

#### 4.2.1 Developer Mode
- 完整显示所有分类和字段
- 显示所有秘密信息、命运信息
- 可编辑所有内容

#### 4.2.2 Player Mode
- 默认只显示 basic + knowledge
- 其他分类字段根据配置显示
- 每个字段可单独配置：
  - **可见**: 显示完整内容
  - **部分可见**: 显示自定义文本（如"似乎有不可告人的过去"）
  - **隐藏**: 显示"未知"

#### 4.2.3 可见性配置
- 每个字段单独配置显示方式
- 可保存为模板，应用到同类型角色
- 预设默认配置（如secrets默认隐藏）

### 4.3 关系系统

#### 4.3.1 预设关系类型
10种预设类型：亲属、朋友、恋人、敌人、对手、盟友、师徒、上下级、同事、陌生人

#### 4.3.2 自定义关系类型
- 用户可添加自定义关系类型
- 每种类型可设置名称、图标、颜色

#### 4.3.3 关系属性
- **方向**: 单向(A→B) / 双向(A↔B)
- **描述**: 关系的文字描述
- **数值**: 好感度/信任度等数值（-100到100）
- **历史事件**: 关联的历史事件列表

#### 4.3.4 关系图谱
- 交互式网络图谱
- 节点大小根据关系数量调整
- 连线样式根据关系类型和强度调整
- 支持拖拽移动、缩放平移
- 点击节点显示详情
- 右键菜单：新建关系、编辑、删除
- 筛选：按关系类型、强度筛选

### 4.4 AI分步生成

#### 4.4.1 生成步骤
```
步骤1: 基础信息 (basic)
  → 生成姓名、身份、等级、简述

步骤2: 认知信息 (knowledge)
  → 基于基础信息生成性格、外貌、背景、动机

步骤3: 秘密信息 (secrets)
  → 基于前两步生成心理创伤、隐藏身份、秘密

步骤4: 属性信息 (attributes)
  → 生成健康状态、特殊状态

步骤5: 关系信息 (relations)
  → 推荐与其他角色的关系，用户确认/修改

步骤6: 技能物品 (skills/inventory)
  → 生成技能和持有物品
```

#### 4.4.2 迭代优化
- 每个步骤生成后可以：
  - **接受**: 保存当前结果，进入下一步
  - **重新生成**: 用相同参数重新生成
  - **修改提示词**: 调整提示词后重新生成
  - **迭代优化**: 提出修改意见，AI根据意见调整
  - **AI质量检查**: 检查一致性、完整性、质量评分

#### 4.4.3 生成参数
| 参数 | 说明 | 必填 |
|------|------|------|
| 角色类型 | 主角/配角/反派/NPC | 是 |
| 角色概念 | 角色的基本概念描述 | 是 |
| 详细程度 | 生成内容的详细程度 | 可选 |
| 创作风格 | 文学/通俗/学术/史诗 | 可选 |
| 世界观参考 | 是否自动关联世界书 | 可选 |

#### 4.4.4 AI质量检查（可选）
- 一致性检查：与世界观是否冲突
- 完整性检查：信息是否完整
- 合理性检查：性格/背景/动机是否合理
- 质量评分：0-100分
- 改进建议

### 4.5 世界书关联

#### 4.5.1 关联方式
- **手动选择**: 从世界书列表中选择关联条目
- **AI推荐**: 根据角色背景自动推荐关联条目
- **双向显示**: 角色页面显示关联条目，条目页面显示关联角色

#### 4.5.2 关联管理

- 查看关联条目详情
- 关联列表排序（按相关度）

### 4.6 列表与筛选

#### 4.6.1 视图模式
- **卡片视图**: 显示头像/图标、名称、类型、简述
- **表格视图**: 显示详细属性列表

#### 4.6.2 筛选功能
- 按类型筛选（主角/配角/反派/NPC）
- 按标签筛选
- 按状态筛选（活跃/死亡/失踪）
- 全文搜索（姓名、背景、描述）

### 4.7 批量操作
