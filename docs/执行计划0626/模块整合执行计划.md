# Storyworks 多模块整合执行计划

> 制定日期：2026-06-26
> 目标：将四个独立模块整合为统一应用，保持模块独立性的同时实现统一管理

---

## 一、整合目标

### 1.1 核心需求

- ✅ 统一启动入口（一个命令启动整个系统）
- ✅ 单一前端应用（内部模块独立但统一配置）
- ✅ 统一后端服务（模块功能独立但共享基础设施）
- ✅ 模块仍可独立使用

### 1.2 技术决策

| 决策项     | 选择                                 |
| ---------- | ------------------------------------ |
| 前端导航   | 左侧边栏导航                         |
| 项目选择   | 全局项目选择器（顶部栏）             |
| 跨模块联动 | 开发时自动选择最佳方式               |
| API风格    | RESTful:`/api/{module}/{resource}` |
| 数据库     | 统一数据库，模块共享表               |
| 配置管理   | 统一config.json，各模块读取          |

### 2.2 数据库设计（统一SQLite）

```sql
-- 项目表
CREATE TABLE projects (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 世界书分类表
CREATE TABLE worldbook_categories (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL,
    parent_id TEXT,           -- 父分类ID，NULL表示顶级
    name TEXT NOT NULL,
    description TEXT,
    template TEXT,            -- 默认条目模板（JSON）
    custom_fields TEXT,       -- 自定义字段定义（JSON）
    sort_order INTEGER DEFAULT 0,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 世界书条目表
CREATE TABLE worldbook_entries (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL,
    category_id TEXT,
    title TEXT NOT NULL,
    content TEXT,             -- Markdown正文
    importance INTEGER DEFAULT 3,  -- 重要度1-5
    visibility TEXT DEFAULT 'public',  -- public/private
    status TEXT DEFAULT 'draft',   -- draft/complete
    tags TEXT,                -- JSON数组
    metadata TEXT,            -- JSON: 创建者等
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    version INTEGER DEFAULT 1,
    FOREIGN KEY (project_id) REFERENCES projects(id),
    FOREIGN KEY (category_id) REFERENCES worldbook_categories(id)
);

-- 世界书关联表
CREATE TABLE worldbook_relations (
    id TEXT PRIMARY KEY,
    source_id TEXT NOT NULL,
    target_id TEXT NOT NULL,
    relation_type TEXT NOT NULL,  -- source/influence/related/oppose
    label TEXT,                   -- 自定义标签
    strength INTEGER DEFAULT 3,  -- 强度1-5
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (source_id) REFERENCES worldbook_entries(id),
    FOREIGN KEY (target_id) REFERENCES worldbook_entries(id)
);

-- 角色表
CREATE TABLE characters (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL,
    name TEXT NOT NULL,
    character_type TEXT,      -- 主角/配角/反派/NPC
    developer_data TEXT,      -- JSON: developer_mode完整数据
    player_data TEXT,         -- JSON: player_mode数据
    world_entry_ids TEXT,     -- JSON数组: 关联的世界书条目ID
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 角色关系表
CREATE TABLE character_relations (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL,
    source_id TEXT NOT NULL,
    target_id TEXT NOT NULL,
    relation_type TEXT NOT NULL,  -- family/enemy/ally/love/rival
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (source_id) REFERENCES characters(id),
    FOREIGN KEY (target_id) REFERENCES characters(id)
);

-- 剧本表
CREATE TABLE scripts (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL,
    parent_id TEXT,           -- 父节点ID
    node_type TEXT NOT NULL,  -- volume/part/chapter/scene/fragment
    title TEXT NOT NULL,
    content TEXT,             -- 剧本内容
    sort_order INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 剧本关联表（与世界/角色的双向绑定）
CREATE TABLE script_references (
    id TEXT PRIMARY KEY,
    script_id TEXT NOT NULL,
    ref_type TEXT NOT NULL,   -- worldbook/character
    ref_id TEXT NOT NULL,
    FOREIGN KEY (script_id) REFERENCES scripts(id)
);

-- 预设表
CREATE TABLE presets (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL,
    name TEXT NOT NULL,
    description TEXT,
    template TEXT,            -- 预设模板内容
    parameters TEXT,          -- JSON: 可调参数
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 版本历史表（用于版本回溯）
CREATE TABLE version_history (
    id TEXT PRIMARY KEY,
    entity_type TEXT NOT NULL,  -- worldbook_entry/character/script
    entity_id TEXT NOT NULL,
    version INTEGER NOT NULL,
    data TEXT NOT NULL,        -- 快照数据
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 2.3 API路由设计

```
# 项目管理
GET    /api/projects                    # 获取项目列表
POST   /api/projects                    # 创建项目
GET    /api/projects/{id}               # 获取项目详情
PUT    /api/projects/{id}               # 更新项目
DELETE /api/projects/{id}               # 删除项目

# 世界书
GET    /api/projects/{pid}/worldbook/categories        # 获取分类树
POST   /api/projects/{pid}/worldbook/categories        # 创建分类
PUT    /api/projects/{pid}/worldbook/categories/{id}   # 更新分类
DELETE /api/projects/{pid}/worldbook/categories/{id}   # 删除分类
PATCH  /api/projects/{pid}/worldbook/categories/sort   # 排序分类

GET    /api/projects/{pid}/worldbook/entries           # 获取条目列表
POST   /api/projects/{pid}/worldbook/entries           # 创建条目
GET    /api/projects/{pid}/worldbook/entries/{id}      # 获取条目详情
PUT    /api/projects/{pid}/worldbook/entries/{id}      # 更新条目
DELETE /api/projects/{pid}/worldbook/entries/{id}      # 删除条目

GET    /api/projects/{pid}/worldbook/relations         # 获取所有关联
POST   /api/projects/{pid}/worldbook/relations         # 创建关联
DELETE /api/projects/{pid}/worldbook/relations/{id}    # 删除关联

POST   /api/projects/{pid}/worldbook/ai/generate       # AI生成条目
POST   /api/projects/{pid}/worldbook/ai/check          # AI质量检查
POST   /api/projects/{pid}/worldbook/ai/classify       # AI自动分类

# 角色（类似结构）
GET    /api/projects/{pid}/characters                  # 获取角色列表
POST   /api/projects/{pid}/characters                  # 创建角色
GET    /api/projects/{pid}/characters/{id}             # 获取角色详情
PUT    /api/projects/{pid}/characters/{id}             # 更新角色
DELETE /api/projects/{pid}/characters/{id}             # 删除角色

# 剧本（类似结构）
GET    /api/projects/{pid}/scripts                     # 获取剧本树
POST   /api/projects/{pid}/scripts                     # 创建节点
PUT    /api/projects/{pid}/scripts/{id}                # 更新节点
DELETE /api/projects/{pid}/scripts/{id}                # 删除节点

# 预设（类似结构）
GET    /api/projects/{pid}/presets                     # 获取预设列表
POST   /api/projects/{pid}/presets                     # 创建预设
PUT    /api/projects/{pid}/presets/{id}                # 更新预设
DELETE /api/projects/{pid}/presets/{id}                # 删除预设
```

## 五、注意事项

### 5.1 向后兼容

- 保留原有apps目录结构，可选择性删除
- 保留原有启动脚本作为备份

### 5.2 模块独立性

- 虽然整合为统一应用，但代码组织上保持模块边界清晰
- 每个模块的路由、服务、组件仍保持独立目录
- 方便未来可能的模块拆分

### 5.3 渐进式迁移

- 可以先整合世界书模块作为试点
- 验证架构可行后再迁移其他模块
