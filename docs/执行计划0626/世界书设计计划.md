# Storyworks 世界书设计计划

> 制定日期：2026-06-26
> 目标：构建专业级世界书系统，支持复杂世界观管理与AI辅助创作

---

## 一、功能需求汇总

### 1.1 用户选择确认
| 功能点 | 选择 |
|--------|------|
| 分类深度 | 多级树形分类（任意深度） |
| 分类管理 | 树形编辑器（拖拽排序、拖拽移动层级） |
| 分类属性 | 名称 + 描述 + 模板 + 自定义字段 |
| 智能分类 | AI自动分类（根据内容自动归类） |
| 条目结构 | 可配置字段模板（不同分类不同结构） |
| 元数据 | 创建时间、修改时间、创建者、状态、标签、重要度、可见性、版本历史 |
| 正文格式 | Markdown |
| 关联类型 | 分类关联（来源/影响/相关/对立） |
| 关联强度 | 标签 + 强度（1-5） |
| 图谱功能 | 可编辑、可交互（在图谱中创建/删除关联） |
| AI生成 | 按钮点击生成 |
| AI控制 | 提示词 + 生成参数 + 可迭代优化 |
| AI质量检查 | 可选（可直接批准或请求检查） |

## 二、数据结构设计

### 2.1 分类系统

```typescript
interface WorldbookCategory {
  id: string;
  projectId: string;
  parentId: string | null;      // null表示顶级分类
  name: string;
  description: string;
  template: CategoryTemplate;   // 默认条目模板
  customFields: CustomField[];  // 自定义字段定义
  sortOrder: number;
  createdAt: Date;
  updatedAt: Date;
}

interface CategoryTemplate {
  // 不同分类可以有不同的默认字段结构
  sections: {
    id: string;
    title: string;           // 如"定义"、"起源"、"特征"
    placeholder: string;     // 占位提示
    required: boolean;
    aiHint: string;          // AI生成时的提示
  }[];
}

interface CustomField {
  id: string;
  name: string;              // 字段名称
  type: 'text' | 'number' | 'select' | 'multi_select' | 'date' | 'relation';
  options?: string[];        // select/multi_select的选项
  required: boolean;
  defaultValue?: any;
}
```

### 2.2 条目系统

```typescript
interface WorldbookEntry {
  id: string;
  projectId: string;
  categoryId: string;
  title: string;
  content: string;             // Markdown正文
  structuredData: Record<string, any>;  // 结构化字段数据
  
  // 元数据
  importance: 1 | 2 | 3 | 4 | 5;
  visibility: 'public' | 'private';
  status: 'draft' | 'complete' | 'archived';
  tags: string[];
  creator: string;
  
  // 版本控制
  version: number;
  createdAt: Date;
  updatedAt: Date;
  
  // AI相关
  aiGenerated: boolean;
  aiPrompt?: string;          // 生成时使用的提示词
  aiMetadata?: {
    model: string;
    temperature: number;
    generatedAt: Date;
  };
}
```

### 2.3 关联系统

```typescript
interface WorldbookRelation {
  id: string;
  sourceId: string;
  targetId: string;
  relationType: 'source' | 'influence' | 'related' | 'oppose';
  label?: string;              // 自定义标签
  strength: 1 | 2 | 3 | 4 | 5;
  description?: string;
  createdAt: Date;
}
```

### 2.4 分类关联类型定义

| 类型 | 标识 | 说明 | 可视化颜色 |
|------|------|------|-----------|
| 来源 | source | 条目A是条目B的来源/起源 | 蓝色 |
| 影响 | influence | 条目A影响了条目B | 绿色 |
| 相关 | related | 条目A与条目B相关 | 灰色 |
| 对立 | oppose | 条目A与条目B对立 | 红色 |

## 四、功能详细设计

### 4.1 分类管理功能

#### 4.1.1 树形编辑器
- **拖拽排序**: 同级分类间拖拽调整顺序
- **拖拽移动**: 拖拽到其他分类下改变层级
- **右键菜单**: 新建子分类、编辑、删除、复制
- **批量操作**: 选中多个分类进行删除、移动

#### 4.1.2 分类模板系统
```
每个分类可配置:
├── 默认条目模板 (AI生成时参考)
│   ├── 字段列表 (定义/起源/特征/影响...)
│   ├── 每个字段的占位提示
│   └── 每个字段的AI生成提示词
├── 自定义字段
│   ├── 字段名称
│   ├── 字段类型 (文本/数字/下拉/多选/日期/关联)
│   ├── 是否必填
│   └── 默认值
└── 继承设置
    └── 子分类是否继承父分类模板
```

#### 4.1.3 AI自动分类
- 新建条目时，根据标题和内容自动推荐分类
- 分类依据: 关键词匹配 + 语义相似度
- 用户可手动覆盖AI推荐

### 4.2 条目管理功能

#### 4.2.1 条目列表
- 列表视图: 表格展示（标题、分类、重要度、状态、更新时间）
- 卡片视图: 卡片展示（适合浏览）
- 筛选: 按分类、状态、重要度、标签筛选
- 排序: 按标题、时间、重要度排序
- 搜索: 全文搜索（标题、正文、标签）

#### 4.2.2 条目编辑
- **结构化编辑**: 根据分类模板显示结构化字段
- **Markdown编辑**: 正文使用Markdown编辑器
- **关联管理**: 在编辑界面管理关联关系
- **自动保存**: 定期自动保存草稿
- **版本对比**: 查看历史版本差异

#### 4.2.3 版本历史
- 自动记录每次保存的版本
- 版本列表: 显示版本号、保存时间、修改摘要
- 版本对比: 并排对比两个版本的差异
- 版本回退: 回退到指定历史版本

### 4.3 关联与图谱功能

#### 4.3.1 关联类型
| 类型 | 说明 | 可视化 |
|------|------|--------|
| 来源 (source) | A是B的来源/起源 | 蓝色实线箭头 A→B |
| 影响 (influence) | A影响了B | 绿色实线箭头 A→B |
| 相关 (related) | A与B相关 | 灰色虚线 A-B |
| 对立 (oppose) | A与B对立 | 红色实线（无箭头） A×B |

#### 4.3.2 关联属性
- **标签**: 自定义关联说明（如"制度基础"、"政治对立"）
- **强度**: 1-5级，影响图谱中线条粗细
- **描述**: 详细说明关联内容

#### 4.3.3 图谱功能
- **节点**: 每个条目是一个节点，大小根据重要度调整
- **边**: 关联关系是边，颜色/粗细根据类型和强度调整
- **布局**: 自动布局算法（力导向布局）
- **交互**:
  - 点击节点: 显示详情、跳转编辑
  - 拖拽节点: 手动调整位置
  - 右键菜单: 新建关联、删除关联
  - 缩放平移: 浏览大型图谱
- **筛选**: 按分类、类型、重要度筛选显示
- **导出**: 导出为图片或JSON

### 4.4 AI生成功能

#### 4.4.1 生成流程
```
1. 用户点击 [AI生成] 按钮
2. 弹出生成对话框
3. 用户填写/确认:
   - 目标分类 (自动填充当前分类)
   - 条目标题 (可选)
   - 生成提示词
   - 生成参数 (字数、风格、详细度)
4. 点击 [生成]
5. AI生成内容，显示在结果区域
6. 用户选择:
   - [接受] → 保存为新条目
   - [重新生成] → 用相同参数重新生成
   - [修改提示词] → 返回修改
   - [AI质量检查] → 进入质量检查流程
```

#### 4.4.2 生成参数
| 参数 | 说明 | 范围 |
|------|------|------|
| 字数范围 | 生成内容的目标字数 | 100-2000字 |
| 详细程度 | 内容详细程度 | 1-10级 |
| 创作风格 | 文风选择 | 学术/文学/通俗/史诗 |
| 参考已有 | 是否参考同分类已有条目 | 是/否 |

#### 4.4.3 AI质量检查（可选）
```
质量检查流程:
1. 生成完成后，用户可选择 [AI质量检查]
2. AI检查内容:
   - 一致性检查: 与已有世界观是否冲突
   - 完整性检查: 是否缺少必要信息
   - 质量评分: 0-100分
   - 改进建议: 具体的改进方向
3. 显示检查结果
4. 用户选择:
   - [接受当前] → 保存当前版本
   - [应用建议重新生成] → AI根据建议重新生成
   - [关闭] → 返回编辑
```
