# Storyworks 预设设计计划

> 制定日期：2026-06-26
> 目标：构建通用预设系统，支持AI生成、自由编辑、广泛适用、自动整理

---

## 一、核心设计原则

| 原则 | 说明 |
|------|------|
| 简单输入 | 用户给出大概目的描述，AI生成预设 |
| AI生成 | 生成严谨、完整的结构化预设 |
| 自由编辑 | 用户可编辑所有预设内容 |
| 广泛适用 | 可用于各种AI生成场景 |
| AI自动整理 | AI帮助优化、格式化、整理预设 |

## 二、预设结构设计

### 2.1 预设整体结构

```typescript
interface Preset {
  id: string;
  projectId: string;
  name: string;
  description: string;
  
  // 结构化维度配置
  dimensions: PresetDimension[];
  
  // 自定义文本块
  customBlocks: CustomBlock[];
  
  // 组合后的完整提示词
  compiledPrompt: string;
  
  // 应用场景配置
 应用场景: ApplicationScene[];
  
  // 元数据
  tags: string[];
  category: string;
  createdAt: Date;
  updatedAt: Date;
  
  // AI生成相关
  aiGenerated: boolean;
  generationInput?: string;  // 用户的原始描述
}

// 维度配置
interface PresetDimension {
  id: string;
  name: string;              // 维度名称（如：时代风格）
  value: string;             // 维度值（如：古风）
  description: string;       // 详细说明
  examples?: string[];       // 示例
  isCustom: boolean;         // 是否为自定义维度
  order: number;             // 排序
}

// 自定义文本块
interface CustomBlock {
  id: string;
  title: string;             // 块标题
  content: string;           // 块内容
  position: 'before' | 'after' | 'override';  // 插入位置
  order: number;
}

// 应用场景配置
interface ApplicationScene {
  sceneType: string;         // 场景类型（worldbook/character/script/external）
  enabled: boolean;          // 是否启用
  adjustments?: string;      // 场景特定调整
}
```

### 2.2 预设维度系统

#### 基础维度（预设）

| 维度 | 说明 | 示例值 |
|------|------|--------|
| 时代风格 | 内容的时代背景 | 古风、近代、现代、未来、架空 |
| 文化风格 | 文化背景 | 东方、西方、混合、架空 |
| 文风基调 | 整体文风 | 文学、通俗、学术、史诗、轻小说 |
| 情感基调 | 情感色彩 | 严肃、轻松、悲情、热血、治愈 |
| 叙事视角 | 叙述视角 | 第一人称、第三人称、上帝视角 |
| 描写详略 | 描写详细程度 | 简洁、适中、详细、细腻 |

#### 自定义维度
- 用户可添加自定义维度
- 维度名称、说明、示例均可编辑
- 支持拖拽排序

## 三、功能设计

### 3.1 AI生成预设

#### 3.1.1 输入方式

用户输入示例：
```
我想要一个古风武侠小说的写作风格，要有江湖气息，语言偏文学性，
描写要细腻，情感基调偏热血，适合写战斗和恩怨情仇的场景。
```

#### 3.1.2 AI生成流程

```
1. 用户输入大概目的描述
2. AI分析输入，提取关键信息：
   - 时代风格：古风
   - 文化风格：东方（武侠）
   - 文风基调：文学性
   - 情感基调：热血
   - 描写详略：细腻
3. AI生成完整预设，包括：
   - 填充所有维度
   - 生成维度说明和示例
   - 生成自定义文本块（如：武侠专用术语）
4. AI自动整理：
   - 格式化输出
   - 检测维度冲突
   - 补充缺失维度
   - 优化描述语言
```

### 3.3 AI自动整理功能

#### 3.3.1 整理内容

| 整理类型 | 说明 |
|----------|------|
| 格式整理 | 统一格式、修正错误、规范标点 |
| 内容优化 | 完善描述、补充细节、优化语言 |
| 维度补全 | 检测缺失维度，自动补充 |
| 冲突检测 | 检测维度间的冲突（如：古风 + 现代） |
| 一致性检查 | 确保各维度描述一致 |

#### 3.3.2 整理流程

```
1. 用户点击 [AI整理] 按钮
2. AI分析当前预设：
   - 检测格式问题
   - 检测内容问题
   - 检测维度冲突
   - 检测缺失维度
3. AI生成整理建议：
   - 显示问题列表
   - 提供修改建议
4. 用户选择：
   - [全部接受] → 应用所有建议
   - [选择性接受] → 勾选要应用的建议
   - [忽略] → 关闭整理
```

### 3.5 预设组合功能

#### 3.5.1 组合方式

```
预设组合 = 基础预设 + 场景覆盖预设

示例：
基础预设：古风武侠风格
场景覆盖：战斗场景专用

组合结果：
- 时代风格：古风（来自基础）
- 文化风格：东方武侠（来自基础）
- 战斗描写：详细、动作感强（来自场景覆盖）
```

#### 3.5.2 组合规则

1. **基础预设**：提供默认的维度配置
2. **场景覆盖**：覆盖特定场景的维度
3. **优先级**：场景覆盖 > 基础预设
4. **合并策略**：
   - 相同维度：场景覆盖值
   - 不同维度：合并保留
   - 冲突维度：场景覆盖值

## 四、AI Prompt 设计

### 4.1 预设生成 Prompt

```markdown
## 系统提示
你是一个专业的写作风格设计助手，擅长根据用户需求生成严谨、完整的预设配置。

## 用户提示
请根据以下描述生成一个写作风格预设：

**用户描述**: {{user_description}}

## 要求
1. 分析用户描述，提取关键风格要素
2. 为每个维度生成合理的配置
3. 生成维度说明和示例
4. 检测并解决维度冲突
5. 补充缺失的重要维度
6. 生成自定义文本块（如适用）

## 输出格式 (JSON)
```json
{
  "name": "预设名称",
  "description": "预设描述",
  "dimensions": [
    {
      "name": "维度名称",
      "value": "维度值",
      "description": "详细说明",
      "examples": ["示例1", "示例2"]
    }
  ],
  "customBlocks": [
    {
      "title": "文本块标题",
      "content": "文本块内容",
      "position": "before|after|override"
    }
  ],
  "tags": ["标签1", "标签2"],
  "category": "分类"
}
```
```

### 4.2 AI整理 Prompt

```markdown
## 系统提示
你是一个专业的预设优化助手，擅长检测和修复预设配置中的问题。

## 用户提示
请检查并整理以下预设配置：

**当前预设**:
{{current_preset}}

## 检查要求
1. 格式检查：标点、空格、换行
2. 内容检查：描述是否清晰、完整
3. 维度冲突检测：维度间是否存在矛盾
4. 缺失维度检测：是否缺少重要维度
5. 一致性检查：各维度描述是否一致

## 输出格式 (JSON)
```json
{
  "issues": [
    {
      "type": "format|content|conflict|missing|inconsistency",
      "description": "问题描述",
      "location": "问题位置",
      "suggestion": "修改建议"
    }
  ],
  "fixed_preset": {
    // 修复后的完整预设
  }
}
```
```
