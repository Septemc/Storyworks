# Storyworks 剧本设计计划

> 制定日期：2026-06-26
> 目标：构建专业级剧本系统，支持多层级大纲、双向绑定、AI生成

---

## 一、功能需求汇总

### 1.1 用户选择确认
| 功能点 | 选择 |
|--------|------|
| 结构层级 | 总纲→卷→篇章→章节→场景→片段 |
| 世界/角色关联 | 双向绑定（剧本引用世界/角色，世界/角色也显示被引用） |
| AI生成 | 支持（大纲、章节、场景等） |
| 预设应用 | 支持（风格预设影响生成） |

## 二、数据结构设计

### 2.1 剧本节点结构

```typescript
interface ScriptNode {
  id: string;
  projectId: string;
  parentId: string | null;      // 父节点ID，null表示顶级
  
  // 节点类型（对应6个层级）
  nodeType: 'outline' | 'volume' | 'part' | 'chapter' | 'scene' | 'fragment';
  
  // 内容
  title: string;
  content: string;              // 剧本内容（Markdown）
  summary?: string;             // 摘要（用于大纲展示）
  
  // 排序
  sortOrder: number;
  
  // 元数据
  status: 'draft' | 'complete' | 'archived';
  tags: string[];
  importance: 1 | 2 | 3 | 4 | 5;
  
  // 时间信息（故事内时间）
  storyTime?: string;           // 故事内时间
  storyLocation?: string;       // 故事内地点
  
  // 关联引用
  references: ScriptReference[];
  
  // AI生成相关
  aiGenerated: boolean;
  aiPrompt?: string;
  
  // 版本信息
  version: number;
  createdAt: Date;
  updatedAt: Date;
}

// 关联引用
interface ScriptReference {
  id: string;
  refType: 'worldbook' | 'character';  // 引用类型
  refId: string;                        // 引用的ID
  refName: string;                      // 引用的名称（冗余，便于显示）
  description?: string;                 // 引用说明
}
```

### 2.2 剧本层级结构

```
总纲 (outline)
├── 卷一 (volume)
│   ├── 篇章一 (part)
│   │   ├── 章节一 (chapter)
│   │   │   ├── 场景一 (scene)
│   │   │   │   ├── 片段一 (fragment)
│   │   │   │   └── 片段二 (fragment)
│   │   │   └── 场景二 (scene)
│   │   └── 章节二 (chapter)
│   └── 篇章二 (part)
└── 卷二 (volume)
```

### 2.3 每层节点的职责

| 层级 | 说明 | 内容建议 |
|------|------|----------|
| 总纲 (outline) | 整个故事的概述 | 故事主题、核心冲突、结局走向 |
| 卷 (volume) | 大的故事段落 | 阶段目标、主要转折点 |
| 篇章 (part) | 中等段落 | 子情节、阶段性高潮 |
| 章节 (chapter) | 小的叙事单元 | 具体事件、场景组合 |
| 场景 (scene) | 具体场景 | 地点、人物、对话、动作 |
| 片段 (fragment) | 场景内的细节 | 具体描写、对话片段 |

### 2.4 双向绑定设计

  referencedBy: {

  characterId: string;

  }[];
}

## 四、功能详细设计

### 4.1 大纲管理

#### 4.1.1 树形结构
- 支持无限层级嵌套（总纲→卷→篇章→章节→场景→片段）
- 拖拽排序调整顺序
- 拖拽移动改变层级
- 展开/折叠节点

#### 4.1.2 节点操作
- 新建节点（指定层级）
- 编辑节点内容
- 删除节点（级联删除子节点）
- 复制节点（包含子节点）
- 移动节点

#### 4.1.3 筛选功能
- 按层级筛选
- 按状态筛选（草稿/完成/归档）
- 按标签筛选
- 按重要度筛选
- 全文搜索

### 4.2 内容编辑

#### 4.2.1 Markdown编辑器
- 支持标准Markdown语法
- 实时预览
- 快捷键支持
- 自动保存

#### 4.2.2 元数据编辑
- 标题、状态、重要度、标签
- 故事内时间、地点
- 摘要（用于大纲展示）

### 4.3 关联引用

#### 4.3.1 添加引用
- 手动选择世界书条目
- 手动选择角色
- AI推荐引用（根据内容自动推荐）

#### 4.3.2 双向绑定
- 剧本节点显示关联的世界书和角色
- 世界书条目显示被哪些剧本引用
- 角色卡显示在哪些剧本中出场

#### 4.4.3 引用管理
- 查看引用详情
- 取消引用
- 批量管理引用

### 4.4 AI生成功能

#### 4.4.1 生成类型
| 生成类型 | 说明 |
|----------|------|
| 大纲生成 | 根据主题生成完整大纲框架 |
| 卷/篇章生成 | 根据总纲生成卷或篇章 |
| 章节生成 | 根据篇章生成章节内容 |
| 场景生成 | 根据章节生成具体场景 |
| 片段生成 | 根据场景生成细节片段 |
| 内容扩展 | 扩展现有内容 |

#### 4.4.2 生成参数
| 参数 | 说明 |
|------|------|
| 字数范围 | 生成内容的目标字数 |
| 详细程度 | 内容详细程度 |
| 风格预设 | 应用预设模块的风格 |
| 参考上文 | 是否参考前文内容 |
| 参考引用 | 是否参考关联的世界书和角色 |

#### 4.4.3 生成流程
```
1. 选择生成类型
2. 填写/确认生成提示词
3. 设置生成参数
4. 点击 [生成]
5. AI生成内容
6. 用户选择:
   - [接受] → 保存为节点内容
   - [重新生成] → 用相同参数重新生成
   - [修改提示词] → 返回修改
   - [迭代优化] → 提出修改意见
   - [AI质量检查] → 检查一致性、连贯性
```

#### 4.4.4 AI质量检查（可选）
- 一致性检查：与世界观是否冲突
- 连贯性检查：与前文是否连贯
- 角色一致性：角色行为是否符合设定
- 质量评分：0-100分
- 改进建议

### 4.5 导出功能

#### 4.5.1 导出格式
| 格式 | 说明 |
|------|------|
| JSON | 完整数据结构，程序使用 |
| Markdown | 可读性好，文档阅读 |
| 纯文本 | 去除格式，简单阅读 |
| 剧本格式 | 标准剧本格式（场景标题、对话等） |

#### 4.5.2 导出选项
- 导出当前节点
- 导出当前节点及子节点
- 导出整个剧本
- 选择性导出（勾选节点）
